# Query In UI Direction

Query In uses a high-contrast developer-tooling interface: near-black canvas, dark product surfaces, white typography, and electric yellow as the primary action signal.

## Core Tokens

- Canvas: `#0a0a0a`
- Surface soft: `#121212`
- Surface card: `#1a1a1a`
- Surface elevated: `#242424`
- Hairline: `#2a2a2a`
- Primary: `#faff69`
- Text on dark: `#ffffff`
- Body text: `#cccccc`
- Muted text: `#888888`
- Tertiary text: `#5a5a5a`
- Error: `#ef4444`

## Principles

- Anchor every page on the black canvas.
- Use electric yellow for primary actions, section labels, icons, and important metrics.
- Keep cards dark and flat with subtle hairline borders.
- Use Inter for interface typography and JetBrains Mono for SQL/code surfaces.
- Show actual SQL and product UI surfaces instead of abstract illustrations.
- Keep button radius at 8px and content surfaces at 12px.
- Avoid secondary brand accents, gradients, decorative orbs, and drop shadows.
- Future WebGL or canvas work should use black/electric-yellow product surfaces, code-console motion, and subtle technical texture instead of violet/cyan gradients.
